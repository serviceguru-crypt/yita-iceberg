# Incident Response

1. Triage severity: security, data integrity, availability, finance, or UX.
2. Preserve logs and audit records.
3. Stop risky workflows if data integrity is threatened.
4. Communicate affected branches and expected next update.
5. Apply rollback or hotfix.
6. Verify with smoke tests.
7. Write a post-incident note with root cause and follow-up actions.

Do not delete audit logs, orders, stock movements, payments, or reversals.
